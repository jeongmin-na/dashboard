# Samsung AI Experience Group Dashboard 사용자 매뉴얼

## 1. 시작하기

이 문서는 Samsung AI Experience Group Dashboard의 주요 기능과 사용법을 안내합니다. 대시보드는 사용자 활동, 비용, 멤버 관리 등 AI 서비스 운영에 필요한 핵심 지표를 시각적으로 제공하여 효율적인 관리를 돕습니다.

### 주요 기능:

- **Overview**: 전체 서비스 현황을 한눈에 파악할 수 있는 종합 대시보드
- **Members**: 그룹 멤버의 상세 정보 및 사용량 관리
- **Usage**: 사용자별, 기간별 상세 사용량 분석
- **Settings**: 대시보드 및 리포트 관련 설정

## 2. 공통 기능

### 2.1. 사이드바 메뉴

화면 왼쪽에 위치한 사이드바를 통해 각 기능 페이지로 빠르게 이동할 수 있습니다.

- **🔄 사용자 정보 업데이트**: 클릭 시 최신 사용자 정보를 불러와 대시보드 전체 데이터를 새로고침합니다. 진행 상태는 버튼 하단의 파란색 진행 바로 표시됩니다.
- **📊 Overview**: 대시보드 메인 페이지로 이동합니다.
- **⚙️ Settings**: 설정 페이지로 이동합니다.
- **👥 Members**: 멤버 관리 페이지로 이동합니다.
- **📈 Usage**: 상세 사용량 분석 페이지로 이동합니다.

## 3. Overview 페이지

서비스의 전반적인 현황을 파악하고 주요 지표를 모니터링하는 페이지입니다.

### 3.1. 기간 필터링

> **Overview 페이지 상단의 기간 필터 영역**

페이지 상단에서 조회하려는 데이터의 기간을 설정할 수 있습니다.

- **기간 선택 버튼 (1d, 7d, 30d)**: 클릭 시 오늘을 기준으로 각각 1일, 7일, 30일간의 데이터를 빠르게 조회합니다.
- **날짜 직접 지정**: 달력 아이콘을 클릭하여 시작일과 종료일을 직접 선택한 후 [적용] 버튼을 눌러 해당 기간의 데이터를 조회합니다.
- **필터 기간 표시**: 현재 적용된 필터 기간이 YYYY-MM-DD ~ YYYY-MM-DD 형식으로 표시됩니다.

### 3.2. 핵심 지표 카드 (Usage Cards)

선택된 기간 동안의 핵심적인 4가지 지표를 보여줍니다.

- **실제 활동한 사용자 수**: 해당 기간에 한 번이라도 활동 기록이 있는 순수 사용자 수입니다.
- **에이전트 편집 라인 수**: AI 에이전트가 생성하거나 수정한 코드 라인 수입니다.
- **수락된 탭 수**: 사용자가 AI의 코드 제안(탭 자동완성)을 수락한 횟수입니다.
- **채팅 수**: 사용자가 AI와 채팅한 횟수입니다.

### 3.3. 차트 분석

> **다양한 분석 차트를 선택할 수 있는 드롭다운 메뉴**

드롭다운 메뉴를 통해 총 9가지의 다양한 분석 차트를 확인할 수 있습니다.

- **📈 일별 활동 추이**: 기간 내 일별 전체 활동량 변화를 라인 그래프로 보여줍니다.
- **📊 모델 사용 분포**: 어떤 AI 모델이 얼마나 사용되었는지 막대그래프로 보여줍니다.
- **💰 비용 추이**: 일별 발생 비용의 변화를 라인 그래프로 보여줍니다.
- **📊 활동 유형 분포**: Chat, Code Generation 등 활동 유형별 분포를 막대그래프로 보여줍니다.
- **🔥 사용자 활동 히트맵**: 사용자별 활동량을 막대그래프로 비교하여 보여줍니다. (활동량 상위 20명)
  - **사용자 검색**: 특정 사용자 ID를 검색하면 해당 사용자의 요일별 활동 히트수를 막대그래프로 상세 분석할 수 있습니다.
- **📊 비용 효율성 분석**: 사용자별 '요청 수'와 '비용'을 스캐터 플롯으로 표시하여 효율성을 분석합니다.
- **🎯 모델 선호도**: 사용자별로 어떤 모델을 선호하는지 스택 막대그래프로 보여줍니다. (활동량 상위 20명)
- **🏆 사용자별 성과**: 사용자별 요청 수, 비용, 활동 일수 등 종합적인 성과를 막대그래프로 비교합니다.
- **📈 활동 유형 상세**: 활동 유형별 요청 수, 발생 비용, 사용자 수를 상세하게 보여줍니다.

### 3.4. 멤버 현황 카드

> **전체, 활성, 비활성 멤버 현황을 보여주는 카드**

- **전체 멤버**: 그룹에 속한 모든 멤버의 수를 표시합니다.
- **활성 멤버**: 현재 활성 상태인 멤버의 수를 표시합니다.
- **비활성 멤버**: 현재 비활성 상태인 멤버의 수를 표시합니다.

> 💡 **Tip**: 각 카드를 클릭하면 아래의 **'활동 리스트'**가 해당 멤버 유형(전체/활성/비활성)에 맞춰 자동으로 필터링됩니다.

### 3.5. 활동 리스트

선택된 기간 및 멤버 유형에 따른 개별 사용자 활동 기록을 최신순으로 보여줍니다.

- **마지막 활동**: 사용자의 마지막 활동 시간 (e.g., '오늘', '3일 전')
- **사용자**: 활동한 사용자의 이메일 주소. 클릭 시 Usage 페이지로 이동하여 해당 사용자의 상세 로그를 바로 조회할 수 있습니다.
- **상태**: 현재 사용자의 활성/비활성 상태
- **활동 유형**: 마지막 활동의 종류 (e.g., Chat Request, Code Generation)
- **요청 수 / 비용 / 제한**: 사용자의 월간 누적 요청 수, 발생 비용, 사용량 제한(Hard Limit)
- **페이지네이션**: 리스트 하단에서 페이지를 이동하거나, 우측 상단 드롭다운 메뉴에서 페이지당 표시할 항목 수를 조절할 수 있습니다.

## 4. Members 페이지

그룹 멤버의 전체 목록과 개인별 사용량 한도, 비용 등을 관리하는 페이지입니다.

### 4.1. 정보 및 관리 기능

> **리셋 날짜, 검색, CSV 내보내기 기능**

- **Fast Requests 리셋 날짜**: 멤버들의 월간 Premium Requests가 초기화된 마지막 날짜를 표시합니다.
- **검색**: 이름 또는 이메일로 특정 멤버를 검색할 수 있습니다.
- **📊 Export CSV**: 현재 필터링된 멤버 목록을 CSV 파일로 다운로드합니다.

### 4.2. 멤버 목록

- **정렬**: 각 컬럼명(NAME, EMAIL 등)을 클릭하여 오름차순/내림차순으로 목록을 정렬할 수 있습니다.
- **PREMIUM REQUESTS**: 멤버별 월간 누적 요청 수를 의미하며, 사용량에 따라 색상으로 구분됩니다. (많을수록 녹색, 적을수록 적색)
- **SPEND**: 현재까지 누적된 사용 비용입니다.
- **ROLE**: 멤버의 권한 (Admin/Member)을 표시합니다.
- **페이지네이션**: 목록 하단에서 페이지를 이동하거나, 우측 상단 드롭다운 메뉴에서 페이지당 표시할 항목 수를 조절할 수 있습니다.

## 5. Usage 페이지

기간별, 사용자별 상세 사용 로그를 심층적으로 분석할 수 있는 페이지입니다.

### 5.1. 필터 및 검색

- **기간 필터**: Overview 페이지와 동일하게 기간별 조회가 가능합니다.
- **사용자 검색**: 이름 또는 이메일로 특정 사용자를 검색하면, 아래의 **'사용자별 누적 사용량'**과 'All Raw Events' 테이블이 해당 사용자의 데이터만 필터링하여 보여줍니다.

### 5.2. 사용자별 누적 사용량 정보

선택된 기간 동안의 사용자별 활동을 요약하여 보여줍니다.

- **REQUEST**: 해당 기간 동안 발생한 요청 수
- **SPEND**: 해당 기간 동안 발생한 비용
- **LAST ACT**: 해당 기간 내 마지막 활동일

### 5.3. All Raw Events

선택된 기간 동안 발생한 모든 개별 활동 로그를 시간순으로 상세하게 보여줍니다.

- **DATE**: 이벤트 발생 시각
- **USER**: 활동 사용자 이메일
- **KIND**: 활동 유형
- **MAX_MODEL / MODEL**: 사용된 AI 모델 정보
- **COST**: 해당 활동으로 발생한 비용

## 6. Settings 페이지

자동 리포트, 데이터 내보내기 등 대시보드의 자동화 기능을 설정하는 페이지입니다. 각 항목은 아코디언 메뉴로 구성되어 있습니다.

### 6.1. 활성 기간 설정

- **활성 상태 자동 전환**: 설정된 기간 동안 활동이 없는 사용자를 자동으로 '비활성' 상태로 전환하는 기능입니다.
- **비활성 전환 기간**: 비활성 상태로 전환될 기준 기간을 설정합니다. (e.g., 30일)
- **활성 상태 알림**: 비활성으로 전환되기 전 사용자에게 알림을 보낼지 설정합니다.

### 6.2. Email Report 설정

- **자동 리포트 발송**: 대시보드 현황을 정기적으로 이메일로 받아볼 수 있습니다.
- **발송 주기**: 리포트 발송 주기를 설정합니다. (매일, 매주, 매월)
- **발송 시간 설정**: 리포트를 발송할 구체적인 시간을 설정합니다.
- **수신자 그룹**: 리포트를 받을 대상을 설정합니다.
  - **관리자**: 관리자 권한을 가진 멤버들에게 발송
  - **전체 멤버**: 그룹의 모든 멤버에게 발송
  - **비활성 멤버**: 현재 비활성 상태인 멤버들에게만 발송
  - **직접 입력**: 사용자가 직접 이메일 주소를 입력하여 발송
    - 체크박스를 선택하면 이메일 입력 영역이 나타납니다
    - 쉼표(,)로 구분하여 여러 이메일 주소를 입력할 수 있습니다
    - 예시: `example1@samsung.com, example2@samsung.com`

### 6.3. Export Documents 설정

- **자동 백업**: 데이터를 정기적으로 자동 백업합니다.
- **Export 형식**: 내보낼 데이터의 파일 형식을 선택합니다. (CSV, Excel 등)
- **Export 범위**: 내보낼 데이터의 기간을 설정합니다.
- **[📥 지금 Export 하기] 버튼**: 현재 설정으로 즉시 데이터를 내보냅니다.
